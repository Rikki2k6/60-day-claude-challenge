# Day 31 – AI Supply Chain Control Tower

## Overview

For **Day 31** of the **60 Days Claude Challenge**, I built an **AI Supply Chain Control Tower**, an interactive operations management simulator generated using Claude AI.

The simulator places the user in the role of a **Head of Operations**, responsible for monitoring a live supply chain network and making real-time decisions to maintain business performance. Every decision directly impacts key operational KPIs such as service level, inventory health, customer satisfaction, operating cost, transportation efficiency, and overall score.

---

## Features

- Real-time operational alert system
- Multiple difficulty levels
- Interactive KPI dashboard
- Dynamic operational scoring
- Event log for all incidents
- Timer-based simulation
- Decision-driven outcomes
- Performance grading system
- Replayable simulation
- Fully responsive HTML application

---

## Dashboard Metrics

The simulator tracks the following KPIs throughout the session:

- Service Level
- Customer Satisfaction
- Inventory Health
- Transport Efficiency
- Operating Cost
- Revenue Protected
- Operations Score
- Remaining Time

Each operational decision updates these metrics instantly.

---

## Operational Alerts

During the simulation, various supply chain incidents appear, including:

- Port Congestion
- Supplier Delays
- Demand Spikes
- Inventory Shortages
- Machine Failures
- Weather Disruptions
- Customs Delays
- Transportation Breakdowns
- Damaged Shipments

Each alert includes:

- Incident description
- Business impact
- Remaining response time
- Multiple corrective actions

---

## Decision Making

Every alert offers several possible responses, such as:

- Approve Air Freight
- Use Backup Supplier
- Increase Production
- Transfer Inventory
- Reroute Trucks
- Expedite Shipment
- Wait It Out
- Ignore Alert
- Delay Decision

Each choice has different consequences and affects multiple business metrics.
