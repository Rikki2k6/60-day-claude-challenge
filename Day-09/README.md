# Day 7 – Claude Usage Strategy

## Objective

Learn how to use Claude more effectively by identifying the best model and effort settings for different types of tasks.

---

## My Experiment

I used the Claude Usage Strategy prompt and answered a few questions about my workflow and goals.

### Questions & Answers

**Current Situation:** Student

**Primary Activities:**

* Coding
* Content Creation
* Research & Learning
* Personalized Guidance

**Claude Usage:**

* Mainly for the 60-Day Claude Challenge
* Heavy ChatGPT user otherwise

**Preferred Output Style:**

* High-quality responses
* Personalized answers
* Reasonably fast output

---

## Personalized Strategy

Claude recommended:

### Primary Model

**Sonnet** — Best balance of speed, depth, and quality.

### Effort Settings

| Effort Level | Best Use Case                        |
| ------------ | ------------------------------------ |
| Low          | Quick lookups and simple questions   |
| Standard     | Most daily tasks                     |
| High         | Research, challenge tasks, debugging |
| Max          | Complex multi-step reasoning         |

---

## Workflow Recommendations

| Task                   | Recommended Setup |
| ---------------------- | ----------------- |
| 60-Day Challenge Tasks | Sonnet + High     |
| Coding & Debugging     | Sonnet + Standard |
| Research & Learning    | Sonnet + High     |
| Content Writing        | Sonnet + Standard |
| Personalized Guidance  | Sonnet + High     |
| Deep Research          | Opus + High       |

---

## What I Learned

Before this exercise, I assumed using the most powerful model and highest effort level would always produce the best results.

The strategy showed that:

* Sonnet is sufficient for most tasks.
* High effort should be used selectively.
* Opus is best reserved for genuinely complex problems.
* Choosing the right model is part of prompt engineering.

---

## Biggest Insight

The most valuable recommendation was not about models—it was about context.

Claude emphasized that it performs better when given:

* Background information
* Goals
* Experience level
* Preferences

This directly connects with what I learned during Context Engineering.

Better context often improves outputs more than simply switching to a stronger model.

---

## Mistakes to Avoid

* Using Opus for every task
* Always selecting the highest effort level
* Treating Claude exactly like ChatGPT
* Giving little or no context
* Stopping after a single prompt instead of refining through conversation

---

## Conclusion

This experiment taught me that effective AI usage is not about always choosing the most powerful model.

The best results come from matching the right model, effort level, and amount of context to the task.

For my workflow, Sonnet remains the primary choice, while High effort and Opus should be reserved for situations that genuinely require deeper reasoning.
