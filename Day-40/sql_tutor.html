How this was built

System prompt design

The underlying Claude call is scoped narrowly to one job: turn a plain-English table description + question into a correct SQL query with a teaching-first explanation. It's instructed to (1) infer a reasonable schema from informal descriptions rather than demanding DDL, (2) default to standard ANSI SQL unless a dialect is mentioned, (3) always return a fixed JSON shape — sql, explanation, steps[], mistakes[], practice_question, practice_answer — so the UI never has to guess at formatting, and (4) flag ambiguity (e.g. an implied join column) instead of silently assuming it. Off-topic or abusive input gets a short, in-character redirect rather than a hard refusal wall, since the audience is beginners who might genuinely paste unrelated text by mistake.

Why this UI

A generic chat box hides the thing a beginner actually needs to see: the query itself, clearly separated from the reasoning. So the interface splits input (tables + question) from a structured output panel — SQL in a code block, then explanation, then a numbered step-by-step (mirrors how you'd actually reason through a JOIN or GROUP BY), then common mistakes as a distinct warning block, then a practice question with a hidden answer to close the loop on whether the concept landed. The dark, code-editor aesthetic signals "this is a technical tool," not a toy chatbot.

Extending it

Straightforward next steps: (1) Tools — give Claude a code-execution tool (e.g. an in-browser SQLite via sql.js) so it can actually run the generated query against sample data the user pastes in, catching errors before showing them. (2) Memory — persist the table description across a session (localStorage or a backend) so users don't retype it every question, and track which SQL concepts (joins, subqueries, window functions) they've practiced. (3) Multi-step flows — turn this into a structured curriculum: detect the user's level from their question complexity, and sequence practice questions that build on prior ones instead of one-off generations.
