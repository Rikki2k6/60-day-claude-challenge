Foundations — Figure, Axes, and Your First Plots

Difficulty: ● ○ ○ ○ Beginner

The Big Idea Intuition

Every Matplotlib chart lives inside two objects: a Figure (the whole canvas/window — think of it as the picture frame) and one or more Axes (the actual plotting area inside that frame — the drawing itself). A figure can hold many axes (multiple charts side by side); an axes holds exactly one plot.

Analogy: the Figure is a photo album page, and each Axes is one photo slot on that page. You can put one photo (one Axes) or a grid of photos (subplots) on a page.
ApproachHow you write itWhen to use
pyplot (state-based)plt.plot(x, y); plt.show()Quick one-off exploratory plots in a notebook
Object-oriented (OO)fig, ax = plt.subplots(); ax.plot(x, y)Anything reused, multi-panel, or production-quality — recommended default
Common misconception: "pyplot and OO are two different libraries." They aren't — plt.plot() is just a shortcut that quietly creates a Figure/Axes for you. The OO API gives you the same objects explicitly, which is why it scales better to multiple subplots.

Your First Line Plot Example

import matplotlib.pyplot as plt

x = [1, 2, 3, 4, 5]
y = [2, 4, 3, 8, 6]

fig, ax = plt.subplots(figsize=(6, 4))
ax.plot(x, y)
ax.set_xlabel("Day")
ax.set_ylabel("Sales (in ₹1000s)")
ax.set_title("Weekly Sales Trend")
plt.show()

Every chart in this tutorial follows the same 4-step rhythm: create → plot → label → show/save. Internalize this rhythm before moving on.

Scatter and Bar Charts Example

# Scatter — good for relationship between two numeric variables
ax.scatter(x, y, color="tomato", s=60)

# Bar — good for comparing categories
categories = ["Mon", "Tue", "Wed", "Thu", "Fri"]
values = [12, 19, 7, 15, 10]
ax.bar(categories, values, color="steelblue")
Choosing a chart: line = trend over a continuous axis (time). Scatter = relationship/correlation between two variables. Bar = comparison across discrete categories.
Try it yourself
Write code that plots a bar chart of five subjects' marks: Math=88, Physics=75, Chem=92, Bio=67, CS=95. Add a title and a y-axis label "Marks".
Show solution 
subjects = ["Math","Physics","Chem","Bio","CS"]
marks = [88, 75, 92, 67, 95]

fig, ax = plt.subplots()
ax.bar(subjects, marks, color="mediumseagreen")
ax.set_ylabel("Marks")
ax.set_title("Subject-wise Marks")
plt.show()

Key Takeaways

  • Figure = whole canvas; Axes = individual plot area. fig, ax = plt.subplots() is the standard starting line.
  • Prefer the object-oriented API (ax.plot, ax.bar) over bare plt.plot once a script grows beyond a single quick plot.
  • Chart type follows the question: trend → line, relationship → scatter, comparison → bar.

Styling & Communication — Making Charts Readable

Difficulty: ● ● ○ ○ Beginner–Intermediate

Why Styling Matters Intuition

A correct chart that's unreadable is a failed chart. This module covers everything that turns a "plot" into a "figure someone can understand in 3 seconds": colors, line styles, markers, titles, legends, and annotations.

Analogy: if Module 1 taught you to build furniture, this module teaches interior design — same object, but now it actually communicates something to the person walking into the room.

Colors, Line Styles & Markers Example

ax.plot(x, y1, color="#5eead4", linestyle="--", marker="o", label="Product A")
ax.plot(x, y2, color="#7c9cff", linestyle="-",  marker="s", label="Product B")

ax.set_title("Product Comparison", fontsize=14, fontweight="bold")
ax.set_xlabel("Month")
ax.set_ylabel("Revenue (₹L)")
ax.legend(loc="upper left")
ax.grid(alpha=0.3)
ParameterValuesEffect
linestyle'-' '--' ':' '-.'solid / dashed / dotted / dash-dot
marker'o' 's' '^' 'x'circle / square / triangle / x at each data point
colorname, hex, or 'C0'..'C9'line/marker/bar color
alpha0.0 – 1.0transparency

Legends & Annotations Example

# Point out a specific data point
ax.annotate("Peak sales", xy=(4, 8), xytext=(4.3, 8.5),
            arrowprops=dict(arrowstyle="->", color="orange"))

# Legend with a custom location and no border
ax.legend(loc="best", frameon=False)
Common misconception: "the legend appears automatically." It only appears if (a) you passed label= to each plot call and (b) you call ax.legend(). Forgetting either is the #1 cause of "why is my legend empty/missing" questions.

Labeling Checklist Comparison

ElementMethodSkip it and...
Titleax.set_title()Viewer doesn't know what the chart is about
Axis labelsax.set_xlabel/ylabel()Units are ambiguous (₹? %? count?)
Legendax.legend()Multiple series become indistinguishable
Gridlinesax.grid()Harder to read precise values (use sparingly)
Try it yourself
Plot two lines (temperatures for two cities over a week) with different colors, dashed vs solid linestyles, markers, a title, axis labels, and a legend.
Show solution 
days = ["Mon","Tue","Wed","Thu","Fri","Sat","Sun"]
delhi = [38, 39, 41, 40, 37, 36, 38]
shimla = [21, 22, 20, 19, 23, 22, 21]

fig, ax = plt.subplots(figsize=(7,4))
ax.plot(days, delhi, color="tomato", linestyle="-", marker="o", label="Delhi")
ax.plot(days, shimla, color="steelblue", linestyle="--", marker="s", label="Shimla")
ax.set_title("Temperature Comparison: Delhi vs Shimla")
ax.set_xlabel("Day"); ax.set_ylabel("Temp (°C)")
ax.legend()
ax.grid(alpha=0.3)
plt.show()

Key Takeaways

  • Legends need both label= on each series and an explicit ax.legend() call.
  • Title + axis labels + units are non-negotiable for any chart leaving your notebook.
  • Use color/linestyle/marker together (not just color) so charts stay readable in grayscale or for colorblind viewers.

Multi-Panel Layouts & More Chart Types

Difficulty: ● ● ● ○ Intermediate

Subplots: Many Axes, One Figure Intuition

plt.subplots(rows, cols) returns a Figure and a grid of Axes (a NumPy array if more than one). You then plot on each axes individually — this is what powers dashboards.

fig, axes = plt.subplots(2, 2, figsize=(10, 7))

axes[0,0].plot(x, y1)
axes[0,0].set_title("Line")

axes[0,1].scatter(x, y2)
axes[0,1].set_title("Scatter")

axes[1,0].bar(categories, values)
axes[1,0].set_title("Bar")

axes[1,1].hist(data, bins=20)
axes[1,1].set_title("Histogram")

fig.tight_layout()   # prevents titles/labels overlapping
plt.show()
Common misconception: forgetting fig.tight_layout() (or constrained_layout=True) leads to overlapping titles/labels between subplots — this is the most common "why does my subplot grid look broken" issue.

Histograms & Box Plots — Understanding Distributions Example

# Histogram — shape/spread of one numeric variable
ax.hist(marks, bins=10, color="#5eead4", edgecolor="black")

# Box plot — median, quartiles, outliers at a glance
ax.boxplot([group_a, group_b, group_c], labels=["A","B","C"])
ChartAnswers the questionWatch out for
Histogram"What does the distribution look like?"bins count changes the story a lot — try a few values
Box plot"Median, spread, and outliers per group?"Hides multimodal distributions (two peaks look like one box)
Scatter"Do X and Y move together?"Overplotting with large datasets — use alpha

Sharing Axes Across Subplots Example

fig, axes = plt.subplots(1, 2, sharey=True, figsize=(9,4))
axes[0].plot(x, y1); axes[0].set_title("2023")
axes[1].plot(x, y2); axes[1].set_title("2024")
# sharey=True keeps the y-axis scale identical so the two years
# are visually comparable — a subtle but important honesty check
Analogy: sharing axes is like putting two photos in identical frames before comparing them — if one photo is zoomed in and the other isn't, the comparison lies.
Try it yourself
Create a 1×2 subplot grid: left = histogram of 100 random exam scores, right = box plot comparing three sections' scores. Add titles to both.
Show solution 
import numpy as np
scores = np.random.normal(70, 10, 100)
sec_a = np.random.normal(65, 8, 30)
sec_b = np.random.normal(72, 6, 30)
sec_c = np.random.normal(78, 9, 30)

fig, axes = plt.subplots(1, 2, figsize=(10,4))
axes[0].hist(scores, bins=15, color="skyblue", edgecolor="black")
axes[0].set_title("Score Distribution")

axes[1].boxplot([sec_a, sec_b, sec_c], labels=["A","B","C"])
axes[1].set_title("Scores by Section")

fig.tight_layout()
plt.show()

Key Takeaways

  • plt.subplots(rows, cols) gives you a grid of Axes to build dashboards from.
  • Always call fig.tight_layout() when using subplots to avoid overlap.
  • Histogram = distribution shape, box plot = summary stats + outliers per group; pick based on the question, not habit.

Mastery — Pandas Integration, Styling Themes & Export

Difficulty: ● ● ● ● Advanced

Plotting Directly From Pandas Intuition

Pandas' .plot() is a thin wrapper around Matplotlib — it returns an Axes, so you can keep customizing with the same ax. methods you already know. This is the workflow you'll use constantly in real analytics work.

import pandas as pd

df = pd.read_csv("sales.csv")
fig, ax = plt.subplots(figsize=(8,4))
df.plot(x="month", y="revenue", ax=ax, kind="line", color="#5eead4")
ax.set_title("Monthly Revenue")
ax.set_ylabel("Revenue (₹)")
plt.show()
Analogy: df.plot(ax=ax) is like handing pandas your empty canvas (ax) and letting it sketch — then you pick up the pen (ax.set_title, etc.) to finish it.

Styles & Themes Example

print(plt.style.available)   # list built-in themes
plt.style.use("seaborn-v0_8-darkgrid")   # apply globally

# or scope a style to one block only:
with plt.style.context("ggplot"):
    fig, ax = plt.subplots()
    ax.plot(x, y)

Styles set consistent defaults (colors, gridlines, fonts) across every chart in a script — critical for a professional-looking dashboard or report.

Exporting Publication-Quality Images Example

fig.savefig("revenue_chart.png", dpi=300, bbox_inches="tight")
fig.savefig("revenue_chart.svg")   # vector format, infinitely scalable
ParameterWhy it matters
dpi=300Print/high-res quality (default is ~100, fine for screens only)
bbox_inches="tight"Removes excess whitespace around the figure
.svg / .pdfVector formats — scale without pixelation, ideal for reports/slides
Common misconception: "plt.show() saves the image." It only displays it — always use fig.savefig() before or instead of plt.show() if you need a file, and call savefig before show() since some backends clear the figure afterward.

SituationDoAvoid
Many subplots in a loopReuse fig, axes once outside the loopCalling plt.subplots() inside the loop (memory leak)

Try it yourself
Load a pandas DataFrame with columns "month" and "expenses", plot it as a bar chart using the ggplot style, and export it as a 300 dpi PNG named expenses.png.
Show solution 
import pandas as pd
import matplotlib.pyplot as plt

df = pd.DataFrame({
    "month": ["Jan","Feb","Mar","Apr"],
    "expenses": [4200, 3900, 5100, 4700]
})

with plt.style.context("ggplot"):
    fig, ax = plt.subplots(figsize=(7,4))
    df.plot(x="month", y="expenses", kind="bar", ax=ax, legend=False, color="#7c9cff")
    ax.set_title("Monthly Expenses")
    ax.set_ylabel("₹")
    fig.savefig("expenses.png", dpi=300, bbox_inches="tight")
plt.show()

Key Takeaways

  • df.plot(ax=ax) bridges pandas and Matplotlib — same ax. methods apply afterward.
  • Use plt.style.use() for consistent visual themes across a whole report.
  • fig.savefig(dpi=300, bbox_inches="tight") before plt.show() for clean, high-res exports.

Final Practical Challenge

Combine everything you've learned

Build a Mini Sales Dashboard

Using a pandas DataFrame with columns month, revenue, and region, build a single Figure with a 2×2 subplot grid containing:

  • Top-left: line chart of total revenue by month, with title and axis labels
  • Top-right: bar chart comparing total revenue by region
  • Bottom-left: histogram of the revenue distribution
  • Bottom-right: box plot of revenue grouped by region

Apply plt.style.use("seaborn-v0_8-whitegrid"), use fig.tight_layout(), and export the result as dashboard.png at 300 dpi.

Reference solution
Show full solution 
import pandas as pd
import matplotlib.pyplot as plt

df = pd.DataFrame({
    "month": ["Jan","Feb","Mar","Apr","May","Jun"]*2,
    "region": ["North"]*6 + ["South"]*6,
    "revenue": [120,135,110,150,160,145, 90,100,95,120,130,125]
})

plt.style.use("seaborn-v0_8-whitegrid")
fig, axes = plt.subplots(2, 2, figsize=(11, 8))

monthly = df.groupby("month")["revenue"].sum().reindex(
    ["Jan","Feb","Mar","Apr","May","Jun"])
axes[0,0].plot(monthly.index, monthly.values, marker="o", color="#2b6cb0")
axes[0,0].set_title("Total Revenue by Month")
axes[0,0].set_ylabel("Revenue")

by_region = df.groupby("region")["revenue"].sum()
axes[0,1].bar(by_region.index, by_region.values, color=["#2b6cb0","#e07798"])
axes[0,1].set_title("Total Revenue by Region")

axes[1,0].hist(df["revenue"], bins=8, color="#63b3ed", edgecolor="black")
axes[1,0].set_title("Revenue Distribution")

north = df[df.region=="North"]["revenue"]
south = df[df.region=="South"]["revenue"]
axes[1,1].boxplot([north, south], labels=["North","South"])
axes[1,1].set_title("Revenue Spread by Region")

fig.tight_layout()
fig.savefig("dashboard.png", dpi=300, bbox_inches="tight")
plt.show()

Summary Notes

Matplotlib centers on the Figure/Axes model. Master the OO API (fig, ax = plt.subplots()) early — it scales cleanly from a single chart to a full dashboard. Chart choice should follow the question you're answering (trend, relationship, comparison, or distribution). Always label, always tight_layout() with subplots, and always savefig(dpi=300, bbox_inches="tight") for anything leaving your notebook.
